# 06. 桌面端完整 UX Flows（Windows-only / Triptych）

本章只描述 Windows 桌面端的用户可见流程（键鼠习惯、右键菜单、托盘常驻、全局快捷键）。

统一前提（全流程默认成立）：

- 布局为 Triptych 三栏：左栏=Folder 树/导航；中栏=列表/收件箱/搜索结果；右栏=详情/编辑/工作区。
- 关闭按钮默认“最小化到托盘”；真正退出只能托盘右键菜单 `退出`。
- 全局快捷键注册可能静默失败；所有快捷键能力必须在“设置”中可见状态与可配置退路。

流程模板（每条都必须完整）：`入口 → 操作 → 状态反馈 → 异常分支`。

---

## 1) 快速捕捉（全局快捷键 + 托盘入口 + 草稿恢复 + 保存反馈）

- Triptych：左栏保持当前导航上下文；中栏用于承载“收件箱/时间线”的落点与定位；右栏用于可选打开新条目详情。
- 入口：
  - 全局快捷键触发“快速捕捉”快捕窗（默认 `Alt+Space`，可配置）。
  - 托盘右键菜单 `快速捕捉`。
  - 主窗口内主动作（例如工具栏的“快速捕捉”按钮），用于在未启用快捷键时仍可用。
- 操作：
  - 触发后弹出快捕窗并自动聚焦输入框；输入文本。
  - `Enter` 保存并自动隐藏快捕窗回到托盘常驻；`Esc` 取消并隐藏（不保存）。
  - 若存在未提交草稿：快捕窗打开时展示“恢复草稿/丢弃草稿”二选一（默认选中恢复，单次确认即可）。
  - 保存成功后：将新记录落入“收件箱/时间线”（中栏可见），右栏可选自动打开详情（默认不打断：只给轻提示）。
- 状态反馈：
  - 保存中：输入框下方一行轻量状态 `保存中…`，并禁用重复提交（避免连按 Enter 产生重复记录）。
  - 保存成功：轻提示 `已保存` + 可点击动作 `打开详情`（打开后：中栏定位到新条目，右栏显示详情）。
  - 离线：仍允许保存（离线优先），提示 `离线：已保存到本地，联网后将自动同步`。
- 异常分支：
  - 快捷键被占用/注册失败：设置页将该快捷键标记为 `不可用（被系统或其他应用占用）`，并提供“一键修改/恢复默认/禁用快捕快捷键”；托盘菜单入口仍可用。
  - 保存失败（本地落库失败/磁盘只读/存储目录不可写）：提示 `保存失败` + 动作 `复制到剪贴板`（保证文本不丢）+ `打开日志/打开数据目录`。
  - 快捕窗无焦点/被系统拦截置顶（例如 UAC/全屏独占）：提示 `无法获取输入焦点`，提供 `重试` 与 `打开主窗口` 兜底。

## 2) 全局搜索（快捷键 → 搜索框 → 结果列表 → 右栏详情/定位到 folder）

- Triptych：左栏用于展示/定位所属 Folder；中栏展示搜索结果列表；右栏展示详情并支持进一步定位。
- 入口：
  - 主窗口内搜索框（位于中栏顶部，默认聚焦快捷键见“快捷键总表”）。
  - 全局快捷键呼出“全局搜索”（打开/聚焦主窗口并直接聚焦搜索框）。
- 操作：
  - 输入关键词后，按回车或停止输入触发搜索（本地索引优先）；中栏展示结果列表。
  - 结果列表支持键盘上下移动；`Enter` 打开选中项：右栏展示详情并高亮命中片段。
  - 右栏提供次动作 `定位到 Folder`：点击后左栏自动展开到所属 Folder，中栏切换到该 Folder 的列表并定位该条目。
  - 搜索范围默认覆盖：笔记标题/正文片段（若无标题则以首行作为标题展示）、可见的 Folder 名称（用于快速跳转）。
- 状态反馈：
  - 搜索中：中栏顶部显示 `搜索中…`（避免全屏 loading），保留上一次结果直至新结果返回。
  - 无结果：中栏显示空状态 `未找到匹配内容`，提供快捷入口 `清空搜索`。
  - 结果来源：若存在“仅本地/已同步/同步中”差异，可在列表项角标体现（不额外加面板）。
- 异常分支：
  - 本地索引不可用/损坏：提示 `搜索索引需要重建`，提供 `立即重建`（后台执行）并降级为“基础扫描”（结果更慢但可用）。
  - 触发全局搜索快捷键失败：与快速捕捉一致，在设置中显式标记并提供改键；托盘菜单 `显示主窗口` 仍可进入搜索。
  - 结果指向的引用缺失（例如 note_ref 指向的 memo 尚未同步/已被删除）：右栏展示 `内容不可用`，提供 `打开冲突/同步状态` 与 `刷新`。

## 3) Triptych 联动（左栏 Folder 树 → 中栏列表 → 右栏详情/编辑）

- Triptych：左栏负责 Folder 树/回收站等导航；中栏负责列表与选择；右栏负责详情/编辑与状态呈现。
- 入口：
  - 用户在左栏点击某个 Folder（Collections 树节点）。
  - 用户在中栏点击某条记录（收件箱/时间线/某 Folder 下的列表项）。
  - 用户通过右键菜单在树节点或列表项上执行常用动作（移动/删除/恢复等）。
- 操作：
  - 左栏选择 Folder：中栏切换到该 Folder 的列表；右栏保持当前详情（默认）或显示“未选中条目”的空态（可配置，默认保持以减少跳变）。
  - 中栏选择条目：右栏展示详情；若进入编辑模式，编辑发生在右栏并在本地自动保存草稿。
  - 列表与详情互相定位：
    - 在右栏点击 `在列表中显示`：中栏滚动定位并选中。
    - 在中栏执行 `打开详情`：右栏聚焦。
- 状态反馈：
  - 右栏编辑保存：显示 `已保存到本地` / `同步中…` / `同步失败` 的细粒度状态（不弹窗）。
  - 中栏分页/加载更多：仅列表底部 `加载中…`，不阻断左/右栏操作。
- 异常分支：
  - 同步失败导致右栏状态卡在 `同步失败`：提供单个可点击动作 `重试同步（Memos）`，并在托盘菜单同步入口保持可用。
  - 本地数据目录迁移中（见 Windows 集成章节）：在三栏顶部显示只读横幅 `正在迁移数据，暂时禁止编辑/拖拽`，迁移完成提示重启。
  - 权限不足/凭据失效（401）：右栏显示 `登录已失效`，提供 `重新登录`（打开设置）并保持离线可编辑（仅标记为待同步）。

## 4) 拖拽整理（memo/note_ref 拖入 folder；hover 展开；边缘滚动；禁止拖入子孙）

- Triptych：左栏作为 Folder 树放置目标；中栏作为拖拽来源（列表）；右栏作为可选工作区/网格放置目标。
- 入口：
  - 中栏列表项（memo 或 note_ref）支持拖拽；左栏 Folder 树节点与右栏工作区（若展示 Folder 网格）作为放置目标。
  - 右键菜单提供同等能力 `移动到…`（给不习惯拖拽的用户）。
- 操作：
  - 用户按住列表项拖动：显示拖拽浮层（drag overlay），原位置保留半透明残影。
  - 拖到左栏树节点上：
    - 悬停 800ms 自动展开该 Folder（hover 展开），以便拖入更深层级。
    - 拖到滚动容器边缘触发边缘滚动（edge scrolling），可拖到长列表/深层树。
  - 松开鼠标完成移动：中栏列表即时移除/重排（乐观更新），右栏若正在展示该条目则保持打开但更新其所属位置提示。
  - 多选拖拽（可选增强）：默认仅支持单条拖拽；若支持多选，则使用 `Ctrl`/`Shift` 选择后拖拽，浮层显示计数 `已选 N 项`。
- 状态反馈：
  - 放置可用：目标 Folder 高亮；放置不可用：光标显示禁止符号，目标不高亮。
  - 移动完成：轻提示 `已移动到 <Folder>`，可点击 `撤销`（短时有效，撤销仅影响本地并写入反向 mutation）。
  - 离线：仍允许整理（结构先落本地 outbox），提示 `离线：结构变更将稍后同步`。
- 异常分支：
  - 禁止拖入子孙/形成环（Folder 拖 Folder 或任意会形成环的结构移动）：直接禁止放下并提示原因 `不能移动到自己的子级`。
  - 拖拽目标无权限/不可写（例如数据迁移中、只读模式、未登录导致结构 API 不可用）：禁止放下并提示 `当前不可整理：请完成登录/等待迁移结束`。
  - 同步冲突导致移动被拒绝（Flow push rejected/conflict）：移动在 UI 保留为“本地待处理”，并在中栏该条目旁标记 `冲突`；点击进入“冲突提示”流程处理。

## 5) 删除/恢复（soft delete、回收站、彻底删除边界与确认）

- Triptych：左栏提供 `回收站` 固定入口；中栏展示当前上下文列表（含回收站列表）；右栏展示详情与删除/恢复动作。
- 入口：
  - 中栏列表项右键菜单：`删除`。
  - 右栏溢出菜单：`删除`。
  - 回收站入口（左栏固定节点 `回收站`）。
- 操作：
  - `删除` 默认执行软删除（soft delete）：条目从当前列表消失，并进入回收站。
  - 在回收站中选择条目：右栏显示删除信息（删除时间、原位置/原 Folder）。
  - `恢复`：将条目恢复到原位置；若原 Folder 已不存在，则恢复到 `收件箱` 并提示需要重新归档。
  - `彻底删除`：仅在回收站内可见；执行前必须二次确认（强调不可恢复），并默认要求用户输入一次确认（例如输入 `DELETE`）以防误触。
- 状态反馈：
  - 删除成功：轻提示 `已移入回收站` + `撤销`（短时有效）。
  - 恢复成功：轻提示 `已恢复` + `在列表中显示`（定位到中栏并选中）。
  - 同步状态：删除/恢复都会产生待同步状态标记（不弹窗）。
- 异常分支：
  - 网络失败/离线：删除/恢复仍可执行（本地标记 + outbox），提示 `离线：将稍后同步`。
  - 恢复时目标冲突（原位置已被占用/排序信息失效/服务端快照不同）：恢复后在中栏标记 `需要整理`，并给出 `打开冲突详情`。
  - 彻底删除失败（文件句柄占用/磁盘错误）：提示 `彻底删除失败`，条目仍保留在回收站并显示原因。

## 6) 分享/导出（ShareCard 图片导出、复制图片/复制文本；失败提示；导出路径权限门）

- Triptych：中栏用于选择要分享的条目；右栏承载 ShareCard 预览与导出面板；左栏保持导航上下文不打断。
- 入口：
  - 右栏溢出菜单：`分享/导出`。
  - 列表项右键菜单：`导出 ShareCard`（不打开详情也可导出，默认仍会在右栏展示预览以便确认）。
- 操作：
  - 打开分享面板（右栏内的轻量抽屉/弹层）：
    - 主动作：`导出图片（ShareCard）`（生成 PNG）。
    - 次动作：`复制图片到剪贴板`、`复制文本`。
  - 导出图片必须走“保存对话框”获得用户明确授权路径；导出后提供 `打开文件所在位置`。
  - 复制图片成功后：给出轻提示并说明目标（例如 `已复制图片，可粘贴到微信/邮件`）。
- 状态反馈：
  - 生成中：右栏显示 `正在生成 ShareCard…`，支持取消。
  - 成功：`已导出`（显示文件名与路径片段）/ `已复制`。
  - 失败：明确失败原因（权限/磁盘空间/路径不可写）并给出可行动作（重试/另存为/复制文本）。
- 异常分支：
  - 导出路径权限门：用户取消保存对话框视为“未导出”，不算失败；保持面板不关闭以便继续选择其他动作。
  - 写盘失败（无权限/磁盘满/文件被占用）：提示 `导出失败`，提供 `选择其他位置` 与 `复制文本` 兜底。
  - 生成失败（渲染异常/内容过长/图片资源未缓存）：提示 `生成失败`，并提供 `仅导出纯文本` 或 `重试（稍后）`。

## 7) 冲突提示（Memos：冲突副本；Flow：rejected/server_snapshot 的呈现与操作）

- Triptych：中栏以状态标记暴露“冲突项”集合；右栏提供冲突解释、对比与处理动作；左栏保持导航上下文并允许从 Folder 维度重新定位。
- 入口：
  - 顶部横幅提示：`检测到冲突，需要处理`（点击打开冲突列表）。
  - 中栏条目状态标记 `冲突`（点击进入右栏冲突详情）。
  - 设置页/托盘菜单可见“同步状态摘要”，若存在冲突则显示 `冲突：N`。
- 操作：
  - Memos（笔记正文）冲突：
    - 系统自动生成“冲突副本”（保留本地改动），原记录在右栏展示为“服务端版本”。
    - 右栏提供对比视图：左侧为“服务端版本”，右侧为“本地冲突副本”（只在该视图内出现，不新增全局页面）。
    - 用户可选动作：`保留两者`（默认）、`将副本内容复制到当前版本`、`删除冲突副本（进入回收站）`。
  - Flow（Todo/Collections 结构）冲突：
    - 当 push 被 rejected 且带 `server_snapshot`/`rejected[].server`：右栏显示“本地操作被拒绝”的说明，并展示“服务端快照”和“本地变更摘要”。
    - 用户可选动作：
      - `应用服务端版本`（恢复一致，放弃本地变更）。
      - `保留本地副本`（以“新条目/新结构项”的方式保存本地内容，避免丢数据）。
      - `强制覆盖（高级）`：仅对允许覆盖的资源开放；执行前二次确认并提示风险（可能覆盖其他设备的更改）。
- 状态反馈：
  - 冲突处理中：右栏显示 `处理中…`，避免重复操作。
  - 处理完成：冲突标记从中栏消失；同步状态回到 `待同步/同步中/空闲`。
  - 处理失败：给出明确失败原因，并提供 `复制变更摘要`（便于人工处理）与 `重试`。
- 异常分支：
  - 网络失败/离线：允许用户先做“本地决策”（保留副本/应用快照到本地），并提示 `离线：处理结果将在联网后同步`。
  - 冲突项引用缺失（例如 note_ref 指向的 memo 仍未拿到 ref_id）：右栏提示 `引用尚未就绪`，提供 `立即同步：笔记（Memos）` 与 `稍后再试`。
  - 连续冲突（同一项多次 rejected）：提示 `检测到重复冲突`，默认切换到“保守模式”（仅应用服务端版本或保留本地副本，不自动强制覆盖）。

## 快捷键总表（默认键位 + 可配置 + 冲突行为）

说明：

- 所有快捷键都必须可在“设置”中修改/禁用/恢复默认。
- 若注册失败（被系统/其他应用占用），必须在设置中显示 `不可用`，并提供替代入口（托盘菜单/应用内按钮）。

| 功能 | 默认键位 | 可配置 | 冲突时行为 | 备注 |
|---|---|---|---|---|
| 显示主窗口 / 聚焦 | `Ctrl+Alt+M` | 是 | 设置中标记不可用；托盘 `显示主窗口` 兜底 | 若已可见则仅聚焦 |
| 隐藏主窗口（最小化到托盘） | `Ctrl+Alt+H` | 是 | 标记不可用；点击窗口 X 仍可隐藏 | 不等价于退出 |
| 快速捕捉 | `Alt+Space` | 是 | 标记不可用；托盘 `快速捕捉` 兜底 | 保存后自动隐藏 |
| 全局搜索（打开并聚焦搜索框） | `Ctrl+Shift+F` | 是 | 标记不可用；主窗口搜索框可用 | 搜索结果在中栏 |
| 立即同步：结构与待办（Flow） | `Ctrl+Alt+S` | 是 | 标记不可用；托盘菜单兜底 | 文案必须区分 Flow |
| 立即同步：笔记（Memos） | `Ctrl+Alt+D` | 是 | 标记不可用；托盘菜单兜底 | 文案必须区分 Memos |
| 打开设置 | `Ctrl+,` | 是 | 标记不可用；托盘 `打开设置` 兜底 | 常见桌面习惯 |
| 切换焦点：左栏 | `Ctrl+1` | 是 | 仅影响应用内 | 键盘优先 |
| 切换焦点：中栏 | `Ctrl+2` | 是 | 仅影响应用内 | 键盘优先 |
| 切换焦点：右栏 | `Ctrl+3` | 是 | 仅影响应用内 | 键盘优先 |
| 列表选择上/下 | `Up` / `Down` | 否 | N/A | 中栏列表默认 |
| 打开选中项详情 | `Enter` | 否 | N/A | 中栏→右栏 |
| 删除选中项（软删除） | `Del` | 是 | 应用内仍可通过右键删除 | 删除进入回收站 |
| 恢复（在回收站） | `Ctrl+R` | 是 | 标记不可用；右栏按钮可用 | 仅在回收站可用 |
| 撤销上一步（如移动/删除） | `Ctrl+Z` | 是 | 应用内可用 | 仅短时可撤销 |
